## 自動投稿パターン（5種類）

### パターン1：朝のスケジュール投稿（毎朝7:30 JST）

**トリガー：** GitHub Actions cron `30 22 * * *`（22:30 UTC）
**データソース：** `data/matches.json` の当日試合を抽出
**投稿条件：** 当日開催試合が1件以上ある場合

**投稿フォーマット：**
```
⚽ 今日の日本人選手の試合（{月}/{日}）

{試合リスト（最大5件）}
例：
🇳🇱 エールディビジ
17:00 AZ vs フェイエノールト
└ 毎熊晟矢（AZ）

🏴󠁧󠁢󠁥󠁮󠁧󠁿 プレミアリーグ
21:00 リバプール vs アーセナル
└ 遠藤航（リバプール）

詳細・中継情報 👇
football-jp.com
#海外サッカー #日本人選手 #{選手名}
```

---

### パターン2：試合結果速報（試合終了後・18時/22時/翌朝8時 JST に確認）

**トリガー：** GitHub Actions cron（18時・22時・翌朝8時）
**データソース：** `data/matches.json` の `status: "FINISHED"` かつ未投稿の試合
**投稿条件：** 試合が終了していてスコアが取得できた場合
**重複防止：** 投稿済み試合IDを `data/x_posted_matches.json` に記録

**投稿フォーマット（勝利）：**
```
✅ 試合結果

🏴󠁧󠁢󠁥󠁮󠁧󠁿 プレミアリーグ
リバプール 2-1 アーセナル

🇯🇵 遠藤航（リバプール）出場

詳細・ハイライト 👇
football-jp.com/results
#プレミアリーグ #遠藤航 #海外サッカー
```

---

### パターン3：ゴール速報（試合中・データ更新タイミング次第）

**トリガー：** GitHub Actionsが試合中に更新された場合（昼12時・夕方18時の更新で検出）
**データソース：** `data/scorers.json` の得点者情報
**投稿条件：** 日本人選手がゴール記録（前回取得との差分）
**注意：** リアルタイムではなく、データ更新タイミングに依存（遅延あり）

**投稿フォーマット：**
```
⚽ ゴール速報！

{選手名} がゴール！！
🇯🇵 {リーグ名} 第{節}節
{ホーム} vs {アウェイ}

今シーズン{N}ゴール目 🔥

football-jp.com/players/{選手ID}
#{選手名} #{リーグ名} #海外サッカー
```

---

### パターン4：週次まとめ（毎週月曜朝9:00 JST）

**トリガー：** GitHub Actions cron `0 0 * * 1`（月曜0:00 UTC）
**データソース：** `data/matches.json` の過去7日間の終了試合
**投稿条件：** 先週に試合が1件以上あった場合

**投稿フォーマット：**
```
📊 先週の日本人選手まとめ（{月}/{日}〜{月}/{日}）

試合数：{N}試合
ゴール：{N}得点（{選手名}ほか）
アシスト：{N}アシスト

今週の注目試合 👇
football-jp.com
#海外サッカー #日本人選手 #週間まとめ
```

---

### パターン5：特別イベント投稿（W杯カウントダウン・FIFAランキング更新等）

**トリガー：** 手動実行 or 特定日付に反応するcron
**種類：**
- W杯開幕カウントダウン（毎月1日）
- FIFAランキング更新日（月2回・木曜）
- W杯組み合わせ抽選日（2025年12月予定）
- 大会節目（グループステージ突破・ベスト16等）

**W杯カウントダウン例：**
```
🏆 W杯2026まで あと{N}日！

🇯🇵 日本の出場選手の多くが
今まさに海外でプレー中です

全選手の試合・配信情報はこちら👇
football-jp.com/worldcup
#W杯2026 #FIFA #日本代表 #海外サッカー
```

## X API v2 制限と設計方針

### 無料プラン（Free tier）の制限
| 項目 | 制限 |
|------|------|
| 月間投稿数 | 1,500ツイート/月 |
| API読み取り | 1,500リクエスト/月（制限あり） |
| アプリ数 | 1アプリ |
| v2エンドポイント | `POST /2/tweets` のみ（書き込み） |

### 想定月間投稿数
| パターン | 頻度 | 月間件数 |
|----------|------|---------|
| 朝スケジュール | 試合日のみ（約20日） | 20 |
| 試合結果 | 試合数×1（約60試合） | 60 |
| ゴール速報 | ゴール数（約20） | 20 |
| 週次まとめ | 4回/月 | 4 |
| 特別イベント | 2〜4回/月 | 4 |
| 手動投稿 | 週3〜5回 | 20 |
| **合計** | | **約128投稿** |

**1,500の制限に対して約128投稿 → 余裕あり（余裕率91%）**

## GitHub Actions cron 設計

```yaml
# .github/workflows/x-post.yml 参照
schedule:
  - cron: '30 22 * * *'   # 朝7:30 JST → 今日のスケジュール投稿
  - cron: '0 9 * * *'     # 夕方18:00 JST → 昼試合の結果投稿
  - cron: '0 13 * * *'    # 夜22:00 JST → 夜試合の結果投稿
  - cron: '0 23 * * *'    # 翌朝8:00 JST → 深夜試合の結果投稿
  - cron: '0 0 * * 1'     # 月曜9:00 JST → 週次まとめ
```

## エラーハンドリング・フォールバック

### API呼び出し失敗時
1. 最大3回リトライ（指数バックオフ：30秒→60秒→120秒）
2. 3回失敗後、GitHub Actionsのログにエラー記録
3. 必要に応じてメールまたはDiscord通知（NOTIFY_WEBHOOK設定で）

### データが取得できない場合
- 当日試合データが空 → 投稿スキップ（エラーではない）
- JSON解析失敗 → エラーログ記録・投稿スキップ

### 投稿重複防止
- 投稿済み試合IDを `data/x_posted_matches.json` に保存
- Actions実行時に読み込み、既投稿はスキップ
